1. 시스템 구성 개요

클라이언트(WebView) - 프록시 서버(Node.js/Express) - MySQL(사용자 데이터) / Redis(캐시) - Naver API의 구조로 동작한다. 모든 Naver API 요청은 프록시 서버를 경유하며, API 키는 서버 .env에서만 관리된다.
계층구성 요소역할
클라이언트WebView App (React + Flutter)UI 렌더링, GPS 수집, 사용자 인터랙션
프록시 서버Node.js + ExpressAPI 키 보안, CORS, Rate Limit, 데이터 가공, DB 연동
MySQLtbl_users, tbl_user_preferences, tbl_selection_history, tbl_favorites사용자 데이터 영구 저장
RedisRedis 7.xNaver API 응답 캐시, Rate Limit 카운터
외부 APINaver Search / Maps / Reverse Geocoding식당 원천 데이터 실시간 조회

2. SC-B-01 | 앱 시작 및 사용자 식별

최초 실행 시 device_id를 tbl_users에 등록하고, 이후 실행부터는 기존 사용자 데이터(선호도 등)를 로드한다.
Step액터동작비고
1클라이언트앱 실행 - device_id 생성 또는 로컬 저장값 로드UUID v4 형식
2프록시 서버POST /api/users - device_id로 tbl_users UPSERT신규: INSERT / 기존: last_active_at 갱신
3MySQLtbl_users 저장 완료 - user_id 반환
4프록시 서버tbl_user_preferences 조회 - 선호도 로드신규 사용자: 기본값으로 자동 생성
5클라이언트선호도 기반 필터 초기값 설정 후 홈 화면 진입

3. SC-B-02 | 위치 수집 및 식당 조회

Step액터동작비고
1클라이언트GPS 권한 허용 - 위경도 수집navigator.geolocation.getCurrentPosition()
2프록시 서버GET /api/geocode - Naver Reverse Geocoding 호출Redis 캐시 우선 조회
3프록시 서버GET /api/restaurants - 사용자 선호도 파라미터 적용tbl_user_preferences 기반 기본 필터 적용
4Redis캐시 HIT 시 즉시 반환 / MISS 시 Naver API 호출 후 캐시 저장TTL: 30분
5클라이언트식당 목록 표시

4. SC-B-03 | 식당 선택 및 이력 저장

식당 선택 시 tbl_selection_history에 이력을 저장하여 개인화 추천의 기반 데이터로 활용한다.
Step액터동작비고
1클라이언트식당 카드 탭 - 선택 이력 전송 요청POST /api/history
2프록시 서버tbl_selection_history INSERTuser_id, place_id, place_name, category, region, selected_at
3프록시 서버네이버 플레이스 딥링크 URL 반환클라이언트가 딥링크 호출
4프록시 서버비동기: 카테고리별 선택 빈도 집계 후 tbl_user_preferences 갱신선호 카테고리 자동 업데이트

5. SC-B-04 | 즐겨찾기 추가/삭제

Step액터동작비고
1클라이언트즐겨찾기 버튼 탭POST /api/favorites
2프록시 서버tbl_favorites INSERT - uq_user_place 중복 체크중복 시 409 Conflict 반환
3aMySQLINSERT 성공 - 201 Created 반환
3bMySQL중복 시 DELETE - 즐겨찾기 해제토글 방식: 이미 있으면 삭제
4클라이언트즐겨찾기 아이콘 상태 토글 (채움/빈)

6. SC-B-05 | 에러 처리 및 Rate Limit

Naver API 쿼터 초과 시 stale 캐시를 우선 반환하여 서비스 연속성을 유지한다.
Step액터동작비고
1Naver API일일 쿼터 초과 - 429 Too Many Requests
2프록시 서버Redis stale 캐시 반환 (TTL 만료 후 24시간 이내)
3프록시 서버캐시도 없을 경우 503 반환{ error: "service_unavailable" }
4프록시 서버Rate Limit: 동일 IP 분당 30회 초과 시 429 반환Redis ratelimit 키 기반
