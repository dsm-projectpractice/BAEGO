이슈 관리 및 해결 과정

* 발생한 이슈 (Issue):

  - 모바일 브라우저(Safari, Chrome)에서 전체 화면 레이아웃에 `height: 100vh`를 적용했을 때, 브라우저 하단의 네비게이션 바 때문에 페이지 최하단 UI(버튼 등)가 가려지는 현상 발생.


* 원인 분석:

  - 모바일 브라우저의 `vh` 단위는 상단 주소창과 하단 네비게이션 바의 동적 숨김/나타남을 고려하지 않고 물리적 디스플레이 전체 높이를 계산하기 때문에 발생함.


* 해결 과정 및 결과 (Resolution):

  - 최신 CSS 스펙인 `dvh` (Dynamic Viewport Height) 단위를 도입하여 해결 시도.

  - Tailwind 설정 파일에 `height: '100dvh'` 유틸리티 클래스를 추가 확장.

  - 최상위 래퍼(Wrapper) 컴포넌트에 `h-screen-dvh` 클래스를 적용하여 모바일 브라우저의 UI 바 상태에 맞게 높이가 동적으로 조절되도록 수정함. (MDN CSS Values 참조)

  - 결과: Safari 브라우저에서 하단 버튼이 정상적으로 화면 내에 배치됨.
