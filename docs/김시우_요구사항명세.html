기술적 리스크 및 해결 전략

구분리스크해결 전략
초기화 순서사용자 등록(POST /api/users) 전에 다른 API 요청이 발생하여 user_id 미설정 상태 오류앱 진입 시 userStore 초기화를 await로 처리. user_id가 설정된 이후에만 하위 API 요청 허용
즐겨찾기 동기화홈 화면 카드 아이콘과 즐겨찾기 탭 목록 간 상태 불일치favoritesStore를 단일 진실 공급원으로 사용. 홈 카드 즐겨찾기 아이콘은 favoritesStore.favorites를 구독
GPS 권한 UX진입 시 즉시 권한 요청으로 사용자 이탈 및 브라우저 정책 제한권한 요청 시점을 사용자 액션 이후로 설정, GPS 거부·타임아웃 시 수동 주소 입력 fallback UI 제공
WebView 딥링크외부 앱 호출 시 intent/url scheme 처리 미흡으로 앱 미설치 상황 대응 실패Android는 intent://, iOS는 universal link 방식으로 분기 처리, fallback URL로 모바일 웹 연결 설정
API 응답 오류네트워크 불안정 또는 API 장애 시 빈 화면 노출Axios interceptor 기반 오류 처리, 재시도(retry) 로직 및 스켈레톤 UI 적용으로 사용자 경험 유지
상태 관리필터·위치·추천 데이터 간 상태 불일치Zustand 중앙 스토어로 전역 상태 단일화, 액션 기반 상태 업데이트 패턴 강제
