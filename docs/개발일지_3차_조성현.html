[스크린샷 1] animate-pulse를 활용한 RestaurantCardSkeleton 컴포넌트 구현 코드

// src/components/BottomSheet.tsx (핵심 부분) import { motion, useDragControls } from 'framer-motion'; export const BottomSheet = ({ isOpen, onClose, children }) => { const dragControls = useDragControls(); return ( <motion.div drag="y" dragControls={dragControls} dragConstraints={{ top: 0 }} dragElastic={0.2} onDragEnd={(_, info) => { if (info.offset.y > 100) onClose(); // 100px 이상 드래그 시 닫힘 }} initial={{ y: '100%' }} animate={{ y: isOpen ? 0 : '100%' }} transition={{ type: 'spring', damping: 30, stiffness: 300 }} className="fixed bottom-0 left-0 right-0 bg-white rounded-t-3xl shadow-2xl p-6 z-50" > {children} </motion.div> ); };

이슈 관리 및 해결 과정

* 발생한 이슈 (Issue):

  - iPhone 15 Safari에서 하단 홈바(Home Indicator) 영역과 BottomNav가 겹쳐 탭 아이콘 일부가 가려지는 현상 발생. Android Chrome에서는 정상이나 iOS Safari에서만 재현됨.


* 원인 분석:

  - iOS Safari는 Safe Area 개념을 적용하여 홈바 영역만큼 추가 여백이 필요함. CSS `padding-bottom`에 고정값을 사용하고 있어 Safe Area 높이를 반영하지 못한 것이 원인.

  - HTML `<meta name="viewport">`에 `viewport-fit=cover` 설정이 누락되어 브라우저가 Safe Area 환경변수(`env()`)를 활성화하지 않았음.


* 해결 과정 및 결과 (Resolution):

  - `index.html` viewport 메타 태그에 `viewport-fit=cover` 추가.

  - BottomNav 컴포넌트의 `padding-bottom`을 고정값에서 `env(safe-area-inset-bottom)`으로 교체. Tailwind에서는 임의값(arbitrary value) 문법으로 적용.

    예) `pb-[env(safe-area-inset-bottom)]` 또는 CSS 변수로 추출하여 인라인 스타일 적용.

  - 결과: iPhone 15 Safari에서 홈바 영역 침범 없이 BottomNav가 정상적으로 표시됨. (MDN CSS env() 함수 문서 참조)

* Problem (문제점/아쉬운 점):

  - 햅틱 피드백(`navigator.vibrate()`)이 iOS Safari에서 지원되지 않아 Android 전용으로만 동작 중. 플랫폼 감지 분기 처리가 추가로 필요한 상황.

  - 햅틱 피드백은 `navigator.vibrate` 지원 여부를 런타임에 감지하여 Android에서만 활성화되도록 조건부 처리 추가.
